The CLI is the local and Action entrypoint for Pipr.

```bash
pipr <command> [options]
```

Run `pipr <command> --help` for command-specific help.

## `pipr version`

Print the installed CLI version:

```bash
pipr version
pipr --version
```

The output is plain semver, such as `0.2.2`.

## `pipr update`

Update a CLI installed from the compiled GitHub Release binary:

```bash
pipr update
```

The command checks the latest release, downloads the matching binary only when a newer version exists, verifies the release checksum, validates the downloaded binary with `--version`, and replaces the current executable. If the installed version is already current, the command exits without replacing it.

`pipr update` does not update package-manager installs. For npm or Bun installs, use:

```bash
npm install -g @usepipr/cli@latest
bun install -g @usepipr/cli@latest
```

`pipr update` updates only the local CLI executable. To update Pipr in GitHub Actions, change the workflow pin, such as `uses: somus/pipr@v0.2.2`.

## `pipr init`

Create editable TypeScript config:

```bash
pipr init
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--config-dir <dir>` | `.pipr` | Config directory to create. |
| `--adapters <adapters>` | `github` | Adapter files to initialize. Use `none` to skip workflow files. |
| `--recipe <recipe>` | none | Starter recipe id. |
| `--minimal` | false | Scaffold only `.pipr/config.ts` without `package.json` or `tsconfig.json`. |
| `--force` | false | Overwrite existing Pipr files. |

Examples:

```bash
pipr init --adapters none
pipr init --recipe security-sast
pipr init --minimal
pipr init --config-dir .config/pipr
```

See [Init Recipes](/docs/recipes) for recipe ids.

## `pipr action`

Run inside the GitHub Docker Action:

```bash
pipr action
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--config-dir <dir>` | `.pipr` | Config directory to load. |

`pipr action` expects GitHub Actions environment variables such as `GITHUB_EVENT_PATH` and `GITHUB_WORKSPACE`. Most users call it through `uses: somus/pipr@...`, not directly.

## `pipr check`

Type-load config and validate the runtime plan:

```bash
pipr check
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--config-dir <dir>` | `.pipr` | Config directory to load. |
| `--require-env` | false | Fail when configured provider env vars are missing. |

Use `--require-env` when validating a local environment that should be able to run Pi.

## `pipr dry-run`

Load config and a GitHub event without publishing:

```bash
pipr dry-run --event path/to/event.json
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--event <path>` | required | GitHub event JSON path. |
| `--config-dir <dir>` | `.pipr` | Config directory to load. |

Use this for Action event parsing and trusted config loading. Use `pipr review` for normal local review runs.

## `pipr inspect`

Print the loaded runtime plan:

```bash
pipr inspect
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--config-dir <dir>` | `.pipr` | Config directory to load. |

The output includes models, agents, tasks, commands, tools, publication settings, checks, and limits.

## `pipr skill`

Print or materialize the Pipr setup skill bundled with this CLI release:

```bash
pipr skill
pipr skill path
```

Subcommands:

| Command | Purpose |
| --- | --- |
| `pipr skill` | Print the bundled setup skill and its references to stdout. |
| `pipr skill path` | Materialize the setup skill under the local Pipr skill cache and print the directory path. |

The bundled skill is version-matched to the CLI. Use `pipr skill` when an AI agent needs to install Pipr, interview for repository review policy, create a custom `.pipr/config.ts`, and validate the setup.

## `pipr review`

Run configured change-request review tasks locally without publishing:

```bash
pipr review --base origin/main
```

Options:

| Option | Default | Purpose |
| --- | --- | --- |
| `--base <sha>` | required | Base commit, branch, or ref. |
| `--head <sha>` | working tree | Head commit, branch, or ref. Omit to include staged and unstaged tracked-file edits. |
| `--config-dir <dir>` | `.pipr` | Config directory to load. |
| `--pi-executable <path>` | `pi` on PATH | Pi executable for local runs. |
| `--json` | false | Print structured local review output. |

See [Local Runs](/docs/guide/local-runs) for behavior differences between local and GitHub Action runs.
