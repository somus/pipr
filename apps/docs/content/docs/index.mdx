Pipr runs repository-owned AI review policy on GitHub, GitLab.com, Azure DevOps Services, and Bitbucket Cloud. You define models, review tasks, commands, and publication limits in `.pipr/config.ts`; Pipr builds the diff, runs Pi, validates findings, and publishes through the selected code host.

## Choose a setup path

Start with the delivery path for your code host. The review configuration stays provider-neutral.

| Code host | Change request | Setup |
| --- | --- | --- |
| GitHub | Pull request | [Run the GitHub quickstart](/docs/guide/quickstart) |
| GitLab.com | Merge request | [Configure GitLab CI and webhooks](/docs/guide/gitlab) |
| Azure DevOps Services | Pull request | [Configure an Azure service-hook runner](/docs/guide/azure-devops) |
| Bitbucket Cloud | Pull request | [Configure a Bitbucket webhook runner](/docs/guide/bitbucket) |

Use [local review](/docs/guide/local-runs) when you want to validate a config and inspect review output without publishing to a code host.

## How Pipr works

Every run follows the same provider-neutral path.

```text
Entrypoint
  -> Review task
  -> Diff Manifest
  -> Pi agent
  -> Validated review output
  -> Code host publication
```

| Primitive | Responsibility |
| --- | --- |
| Entrypoint | Starts a task from a change request event, `@pipr` command, or local review. |
| Review task | Gathers context, calls agents, and returns one task result. |
| Diff Manifest | Represents changed files, hunks, and valid inline ranges. |
| Pi agent | Runs model instructions and tools inside the bounded review environment. |
| Review output | Contains a main comment, inline findings, a command response, or a check result. |
| Publication | Applies validation, caps, deduplication, stale-head checks, and native host writes. |

Read [How Pipr works](/docs/concepts) for the domain model, [Runtime flow](/docs/concepts/runtime) for execution details, and [Trust and security](/docs/concepts/trust-security) for config and workspace boundaries.

## Build your review policy

Start with the smallest authoring surface that matches your goal.

| Goal | Use |
| --- | --- |
| Add one general reviewer | [`pipr.review(...)`](/docs/guide/quickstart) |
| Tune models, paths, limits, and checks | [Configuration](/docs/guide/configuration) |
| Generate a focused workflow | [Recipes](/docs/recipes) |
| Compose tasks, agents, and comments | [Custom tasks](/docs/guide/custom-tasks) |
| Look up exact types and flags | [Reference](/docs/reference) |

Pipr owns diff construction, Pi invocation policy, structured-output validation, and code-host writes. Review tasks cannot publish arbitrary comments, parse raw host payloads, or mutate the Pi workspace.
